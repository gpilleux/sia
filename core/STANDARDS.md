# SIA Standards & Detection Rules
**Project Type Definitions and Configuration Schema**

## PROJECT TYPES

SIA recognizes the following project archetypes. Each type triggers specific behavior in the sub-agents.

### `python-fastapi-ddd`
- **Markers**: `pyproject.toml` (fastapi), `domain/` directory.
- **Architecture**: Domain-Driven Design.
- **Agents**: SIA (DDD Specialist), Repository Guardian (Python/Radon).
- **Skills**: `check_complexity.sh` (radon), `check_coverage.sh` (pytest-cov).

### `node-react`
- **Markers**: `package.json` (react, vite/next).
- **Architecture**: Component-based.
- **Agents**: Design Skills (UX/UI).
- **Skills**: `eslint`, `prettier`.

### `generic`
- **Markers**: None of the above.
- **Architecture**: Unstructured.
- **Agents**: Standard set.

## DETECTION RULES

### Git Identity
- **Command**: `git config --get remote.origin.url`
- **Regex**: `.*/(.*)\.git`

### Tech Stack
- **Python**: `test -f pyproject.toml || test -f requirements.txt`
- **Node**: `test -f package.json`
- **Go**: `test -f go.mod`

### Architecture
- **DDD**: `test -d domain && test -d infrastructure`
- **Clean**: `test -d core && test -d data`
- **Django**: `test -f manage.py`
