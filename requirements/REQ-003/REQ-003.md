## REQUIREMENT DESCRIPTION

Implementar un sistema de renderizado dinámico de UI que permita:

1. **Dashboard Basal Dinámico**: Cuando el usuario ingresa a la plataforma, visualiza automáticamente un layout con datos extraídos de documentos procesados (tablas, gráficos, métricas clave, diagramas).

2. **Evolución de UI mediante Chat**: Las respuestas del chatbot modifican la interfaz de forma dinámica, insertando/actualizando visualizaciones (charts, tables, timelines, diagrams) en tiempo real.

3. **Integración MCP-UI**: Utilizar el protocolo `UIResource` de `@mcp-ui/client` (React) + `mcp-ui-server` (Python) para standardizar el flujo de generación y renderizado de componentes visuales.

### Problem to Solve

Actualmente:
- Las respuestas del chat son solo texto plano con markdown básico
- No existe un dashboard basal que muestre información al cargar la plataforma
- Las visualizaciones (función calling) no se renderizan como componentes interactivos
- El sistema genera JSON de visualizaciones pero el frontend no las renderiza

**Objetivo**: Transformar Argus en una plataforma de **Document Intelligence** donde la UI evoluciona dinámicamente basada en el contenido documental + interacciones del usuario.

### Expected Value

1. **UX Superior**: Usuario ve insights visuales inmediatamente al entrar
2. **Interactividad**: Gráficos/tablas generados por GPT se renderizan nativamente
3. **Escalabilidad**: Sistema standardizado para agregar nuevos tipos de visualizaciones
4. **Diferenciación**: Plataforma AI-native con UI que se adapta al contenido documental

## INVARIANTS TO SATISFY

- [ ] **UI Evolution**: `ui_state(t+1) = f(ui_state(t), user_action)` - La UI siempre evoluciona deterministicamente
- [ ] **Content Isolation**: Todas las visualizaciones HTML se renderizan en sandboxed iframes (seguridad)
- [ ] **Type Safety**: `UIResource.mimeType ∈ {text/html, text/uri-list, application/vnd.mcp-ui.remote-dom+javascript}`
- [ ] **Bidirectional Comm**: Frontend puede recibir `UIResource` y enviar `UIAction` de vuelta al backend
- [ ] **Declarative Rendering**: Backend genera UI de forma declarativa, frontend renderiza automáticamente basado en `mimeType`

### ⚠️ CRITICAL PREREQUISITE: Google ADK Migration

**HALLAZGO TÉCNICO**: Durante la investigación arquitectónica se descubrió que el sistema actual **NO utiliza Google ADK Runner en producción**. 

**Estado Actual**:
- ✅ `infrastructure/adk/agents/` existe con 10 agentes definidos
- ✅ `PostgresAdkSessionService` implementado correctamente
- ✅ `AdkAgentFactory` funcional en demos
- ❌ **Chat endpoint usa orquestación custom** (`execute_multi_agent_workflow()`)
- ❌ **No usa `Runner.run_async()`** → No tiene acceso a **ADK Artifact Service**

**Implicación para MCP-UI**:
- MCP-UI `UIResource` requiere `google.genai.types.Part` artifacts del ADK
- ADK Artifact Service maneja binary data (HTML, imágenes, PDFs)
- Orchestrator custom NO puede generar artifacts en formato ADK

**Decisión Arquitectónica**: Implementar **FASE 0** para migrar a Google ADK Runner ANTES de integrar MCP-UI.

**Justificación**:
1. **Dependency Chain**: `MCP-UI ← ADK Artifacts ← ADK Runner`
2. **No Re-work**: Migramos una vez, ganamos ambos beneficios
3. **Small Scope**: Orchestrator ya tiene lógica correcta, solo refactor de interfaces
4. **Future-Proof**: ADK es framework oficial de Google para multi-agent systems

### Research Results Summary

**MCP-UI Core Concepts**:
- `UIResource`: JSON object con `uri`, `mimeType`, y contenido (`text`/`blob`)
- 3 tipos de contenido: `text/html` (rawHtml), `text/uri-list` (externalUrl), `application/vnd.mcp-ui.remote-dom+javascript` (remoteDom)
- Security-first: Todo se renderiza en sandboxed iframes
- Bidirectional: `UIAction` permite comunicación UI → Host

**Integration Pattern**:
```typescript
// Frontend
<UIResourceRenderer 
  resource={uiResource} 
  onUIAction={async (action) => {
    if (action.type === 'tool') {
      const result = await callBackendTool(action.payload)
      return { status: 'handled', data: result }
    }
  }}
/>
```

```python
# Backend
from mcp_ui_server import create_ui_resource

ui_resource = create_ui_resource({
    "uri": "ui://argus/chart/bar",
    "content": {
        "type": "rawHtml",
        "htmlString": generate_bar_chart_html(data)
    },
    "encoding": "text"
})
```
