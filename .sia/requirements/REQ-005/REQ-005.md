## PROBLEM DESCRIPTION

### Current State (SIA 1.1.0)

**Architecture**: Static markdown instructions + bash scripts
- `.github/copilot-instructions.md` → GitHub Copilot reads on every chat
- `skills/*.sh` → Bash scripts with hardcoded paths, no error handling
- `agents/*.md` → Markdown templates (no executable logic)
- `auto_discovery.py` → Single-purpose script (no reusable modules)

**Limitations**:
1. **No Knowledge Persistence**: Every Copilot session starts from zero context
2. **No Semantic Search**: Can't find "all functions that validate email" without grep
3. **Fragile Skills**: Bash scripts break on missing tools, wrong paths, edge cases (REQ-004 findings)
4. **No Testing**: Zero unit tests, integration tests, or E2E validation
5. **Stateless**: Can't learn from past decisions, requirements, refactorings
6. **Copilot-Dependent**: No standalone capabilities (CLI, API, automation)

### Target State (SIA 2.0)

**Architecture**: Autonomous Python application + enhanced Copilot integration
- **Knowledge Graph** (PostgreSQL + PGVECTOR):
  - AST-indexed codebase (classes, functions, imports, dependencies)
  - Embedded documentation (SPRs, REQs, domain analysis)
  - Hybrid search (semantic similarity + keyword + relationship traversal)
- **Skills API** (Python modules):
  - `sia.domain.skills.complexity.ComplexityAnalyzer` (pure domain logic)
  - `sia.application.usecases.AnalyzeComplexity` (orchestration)
  - `sia.infrastructure.tools.RadonAdapter` (external tool wrapper)
  - `sia.api.endpoints.POST /skills/complexity/analyze` (FastAPI)
- **Multi-Layer Testing**:
  - `tests/domain/` → Pure logic (no I/O, mocked repositories)
  - `tests/application/` → Use case orchestration
  - `tests/integration/` → Real database, real tools (radon, pydeps)
  - `tests/e2e/` → API + Knowledge Graph + Skills pipeline
- **ADK Agents** (LlmAgent + tools):
  - Repository Guardian agent uses Skills API as tools
  - Research Specialist agent queries Knowledge Graph
  - Compliance Officer agent validates against embedded REQ specs
- **CLI + API**:
  - `sia analyze complexity backend/src/` (standalone CLI)
  - `sia server` → FastAPI with SSE streaming for agent workflows
  - GitHub Copilot still enhanced via `.github/copilot-instructions.md`

**Capabilities Unlocked**:
- ✅ "Find all functions similar to UserRepository.save()" → Vector search + AST relationships
- ✅ "Show me the complexity history of checkout_service.py" → Time-series metrics

### Architectural Constraints

**DDD**:
- ∀ entity ∈ domain: entity.pure = True (no I/O, no external deps)
- ∀ repository ∈ infrastructure: implements interface ∈ domain
- ∀ usecase ∈ application: orchestrates domain services + repositories
- Dependency rule: Domain ← Application ← Infrastructure ← API

**SOLID**:
- **SRP**: ComplexityAnalyzer only analyzes complexity (no reporting, no persistence)
- **OCP**: New skills extend BaseSkill without modifying orchestrator
- **LSP**: Any IKnowledgeRepository implementation is swappable
- **ISP**: Separate interfaces for read/write (IKnowledgeReader, IKnowledgeWriter)
- **DIP**: Application depends on abstractions (interfaces), not implementations

**KISS**:
- Start with simplest PGVECTOR setup (no hybrid index optimization)
- Use FastAPI defaults (no custom middleware initially)
- Pure Python AST (no ANTLR, no tree-sitter)
- In-memory caching before Redis/Memcached

**Clean Architecture**:
```
API (FastAPI endpoints, SSE)
  ↓
Application (Use Cases, Agent Tools)
  ↓
Domain (Entities, Value Objects, Repository Interfaces, Services)
  ↑
Infrastructure (PostgreSQL, Radon, Embeddings, File I/O)
```

**Testing**:
- Domain: 100% coverage (pure logic, fast, no mocks needed)
- Application: 90%+ coverage (mocked repositories)
- Infrastructure: 80%+ coverage (real DB via Docker, ephemeral)
- E2E: Critical paths only (agent workflows, SSE streaming)
