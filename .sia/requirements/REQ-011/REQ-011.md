## REQUIREMENT DESCRIPTION

Transformar SUPER_AGENT en **orquestador paralelo de sub-agentes** usando GitHub Copilot CLI (`copilot --agent <name> -p "prompt"`). Habilitar ejecución simultánea de múltiples sub-agentes especializados con comunicación asíncrona vía archivos, progress tracking inteligente, y consolidación de resultados SPR.

### Problem to Solve

**Situación actual**:
- Agentes SIA son archivos markdown (`.sia/agents/*.md`)
- Delegación es conceptual: SUPER_AGENT "simula" sub-agente leyendo `.md` y ejecutando lógica manualmente
- Sub-agentes NO pueden ejecutarse en paralelo (overhead secuencial)
- Sin progress tracking real (SUPER_AGENT no sabe qué hace el sub-agente)
- Overhead cognitivo alto (SUPER_AGENT debe ejecutar lógica de múltiples agentes)

**Descubrimiento 1 (runSubagent - DESCARTADO)**:
- ❌ VS Code Copilot tiene `runSubagent` tool pero **NO permite ejecución paralela**
- ❌ Cada invocación crea sesión secuencial (blocking)
- ❌ No viable para orquestación de múltiples sub-agents simultáneos

**Descubrimiento 2 (Copilot CLI - ✅ SOLUCIÓN)**:
- ✅ `copilot --agent <name> -p "prompt" --allow-all-tools` permite invocación no-interactiva
- ✅ Subprocess spawning → múltiples agentes en paralelo
- ✅ Output capturado vía stdout redirection
- ✅ Logs separados por agente (`--log-dir`)
- ✅ File-based communication (status.yaml + progress tracking)

**Ejemplo de delegación actual (simulada)**:
```markdown
User: "Investiga patrones de vector search con pgvector"
→ SUPER_AGENT lee agents/research_specialist.md
→ SUPER_AGENT simula research protocol manualmente
→ SUPER_AGENT invoca mcp_deepwiki_ask_question
→ SUPER_AGENT sintetiza findings
```

**Ejemplo de delegación propuesta (paralela vía CLI)**:
```python
User: "Investiga pgvector patterns + Audita DDD compliance"
→ SUPER_AGENT analiza: 2 tareas independientes
→ SUPER_AGENT spawns paralelos:
    subprocess.Popen(['copilot', '--agent', 'research-specialist', '-p', 'Research pgvector...'])
    subprocess.Popen(['copilot', '--agent', 'repository-guardian', '-p', 'Audit DDD...'])
→ Ambos agentes ejecutan simultáneamente
→ SUPER_AGENT polls status.yaml cada 2s:
    [research-specialist] 45% - "Executing MCP query: langchain-ai/langchain"
    [repository-guardian] 60% - "Running check_complexity.sh"
→ Agentes completan → output.md con SPR
→ SUPER_AGENT consolida results
→ SUPER_AGENT integra findings en Project SPR
```

### Expected Value

**Productividad (BOOST x10)**:
- ✅ **Paralelización real**: 3 sub-agents ejecutan simultáneamente (vs secuencial)
- ✅ **Progress tracking**: SUPER_AGENT monitorea estado en tiempo real (no black box)

## PROGRESS TRACKING PROTOCOL

**CRITICAL**: Update status file every 30 seconds during execution.

```python
import os
import yaml
from pathlib import Path
from datetime import datetime

def update_status(progress: int, task: str, findings_count: int = 0, errors: list = []):
    """Update orchestrator-visible status file"""
    status_file = Path(os.getenv('SIA_STATUS_FILE', '.sia/runtime/status.yaml'))
    if not status_file.exists():
        return  # Not running in orchestrated mode
    
    status = yaml.safe_load(status_file.read_text()) if status_file.exists() else {}
    status.update({
        'state': 'in_progress' if progress < 100 else 'completed',
        'updated_at': datetime.utcnow().isoformat() + 'Z',
        'progress_percent': progress,
        'current_task': task,
        'findings_count': findings_count,
        'errors': errors
    })
    status_file.write_text(yaml.dump(status, default_flow_style=False))
```

**Update points**:
- Before each major step (MCP query, synthesis, validation)
- After completing each finding
- On error (add to errors list)
- On completion (progress=100, state='completed')

- No viable para orquestación de múltiples sub-agents simultáneos

**2. Copilot CLI soporta invocación no-interactiva (ENABLER)**
- `copilot --agent <name> -p "prompt" --allow-all-tools` permite subprocess spawning

- **Decisión**: Approach CLI-based adoptado
