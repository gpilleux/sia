## REQUIREMENT DESCRIPTION

Implementar sistema de skills para lectura de documentos (DOCX, XLSX, PDF) que permita:
1. Usuarios no técnicos extraer texto de archivos sin instalar dependencias manualmente
2. Arquitectura extensible para agregar formatos futuros (CSV, Markdown, etc.)
3. Sincronización automática con `/sync` en nuevos proyectos
4. Zero-friction UX: comando obvio, instalación lazy, error handling robusto

### Problem to Solve

**Usuarios no técnicos** (jefes de proyecto, ingenieros de proceso) necesitan extraer texto de documentos para análisis/procesamiento pero enfrentan:
- Fricción técnica: instalar pandas, openpyxl, PyMuPDF manualmente
- No saben qué librería usar para cada formato
- Scripts existentes no manejan errores (archivos corruptos, password-protected)
- Código duplicado si necesitan múltiples formatos

**Desarrolladores SIA** necesitan:
- Extender formatos sin modificar código existente (OCP)
- Reutilizar readers en múltiples proyectos vía templates
- Testing unitario sin dependencias de `uv`

### Expected Value

**Para usuarios finales**:
- ✅ Comando autoexplicativo: `uv run skills/read_docx.py informe.docx > texto.txt`
- ✅ Zero setup: `uv` instala dependencias automáticamente
- ✅ Error messages claros: "Archivo corrupto", "Password-protected no soportado"

**Para framework SIA**:
- ✅ Extensibilidad: Agregar CSV = crear `csv_reader.py` (3 min)
- ✅ Reusabilidad: Templates copiados en `/sync` a todos los proyectos
- ✅ Mantenibilidad: Core module testeable, facades desacoplados

**Para ecosistema**:
- ✅ 3.4x speedup vs human implementation (19h AI vs 65h Human)
- ✅ Pattern reusable: Strategy + Registry + CLI Facades

## INVARIANTS TO SATISFY

- [x] **∀ reader ∈ Readers**: `reader extends AbstractFileReader ∧ reader.read() → str`
- [x] **∀ facade ∈ {read_docx, read_xlsx, read_pdf}**: `facade.executable = TRUE ∧ facade.shebang = "uv run"`
- [x] **∃! registry**: `registry.auto_discovers(AbstractFileReader.subclasses) = TRUE`
- [x] **file.exists() = FALSE ⇒ exit_code = 1 ∧ stderr ≠ ∅**
- [x] **∀ dependency ∈ {python-docx, openpyxl, PyMuPDF}**: `uv.installs_ephemeral(dependency) = TRUE`
- [x] **output.encoding = UTF-8 ∧ output.format = plain_text**
- [x] **∀ new_format**: `add(new_format) → ∄ modification(existing_readers)`

## TECHNICAL CONTEXT

### Affected Components

- `templates/skills/file_readers/__init__.py`: Registry + auto-discovery
- `templates/skills/file_readers/base.py`: AbstractFileReader (ABC)
- `templates/skills/file_readers/docx_reader.py`: python-docx implementation
- `templates/skills/file_readers/xlsx_reader.py`: openpyxl implementation
- `templates/skills/file_readers/pdf_reader.py`: PyMuPDF implementation
- `templates/skills/read_docx.py`: CLI facade (uv --with python-docx)
- `templates/skills/read_xlsx.py`: CLI facade (uv --with openpyxl)
- `templates/skills/read_pdf.py`: CLI facade (uv --with PyMuPDF)
- `templates/skills/read_file.py`: Universal CLI (power users)
- `installer/install.sh|bat|py`: Copy templates/skills to .sia/skills
- `templates/prompts/sync.prompt.md`: Skills sync logic
- `skills/README.md`: Documentation update

### Technical Dependencies

**Python Libraries** (ephemeral via `uv --with`):
- `python-docx>=1.1.0`: DOCX text extraction (headers, footers, tables)
- `openpyxl>=3.1.0`: XLSX reading (sheets, cells, formulas)
- `PyMuPDF>=1.23.0`: PDF text extraction (OCR-capable, fast)

**Platform**:
- `uv>=0.1.0`: Package manager with ephemeral dependencies
- Python 3.10+: ABC support, `__init_subclass__` for registry

### Architectural Constraints

**DDD Principles**:
- **Bounded Context**: Skills = Infrastructure tools (no domain coupling)
- **Value Objects**: `FilePath`, `ExtractedText` (immutable)
- **Strategy Pattern**: AbstractFileReader = interface, concretes = strategies
- **Repository Pattern**: Registry acts as in-memory repository for readers

**SOLID Principles**:
- **SRP**: Core module (reading logic) vs facades (CLI interface)
- **OCP**: Add CSV reader without modifying existing code
- **LSP**: All readers interchangeable (same interface)
- **ISP**: Minimal interface (`read(filepath) → str`)
- **DIP**: Facades depend on AbstractFileReader, not concretes

**Clean Architecture**:
- **Layer**: Infrastructure (Skills) = outermost layer
- **Dependencies**: Skills → Core (read-only, `UV_STANDARD.md`)
- **No imports**: Skills ¬→ Agents, Skills ¬→ Requirements

**KISS Principle**:
- Plain text output (no complex formats)
- Single responsibility per reader
- Obvious CLI commands (`read_docx` > `extract_text --format docx`)

## ADDITIONAL NOTES

### Usage Examples (No Técnicos)

**Extraer texto de Word**:
```bash
uv run skills/read_docx.py informe_Q4.docx > informe_texto.txt
cat informe_texto.txt
```

**Extraer datos de Excel**:
```bash
uv run skills/read_xlsx.py ventas_2024.xlsx > ventas_raw.txt
```

**Procesar múltiples PDFs**:
```bash
for pdf in reports/*.pdf; do
    uv run skills/read_pdf.py "$pdf" > "txt/${pdf%.pdf}.txt"
done
```

### Usage Examples (Power Users)

**Auto-detectar formato**:
```bash
uv run skills/read_file.py documento.docx
uv run skills/read_file.py datos.xlsx
```

**Listar formatos soportados**:
```bash
uv run skills/read_file.py --list-formats
# Output: docx, xlsx, pdf
```

### Edge Cases

**Archivos password-protected**:
- Exit code 1 + mensaje: "Password-protected files not supported. Decrypt first."

**PDFs escaneados** (sin texto, solo imágenes):
- PyMuPDF with OCR: Requiere Tesseract instalado
- Mensaje: "Scanned PDF detected. Install Tesseract for OCR support."

**Archivos vacíos**:
- Exit code 0 + output vacío (válido)

**Large files** (>100MB):
- openpyxl: Usar `read_only=True` mode
- PyMuPDF: Procesar página por página

### Performance Considerations

**Memory**:
- XLSX: `openpyxl.load_workbook(readonly=True)` → 10x menos RAM
- PDF: Procesar una página a la vez si >1000 páginas

**Speed**:
- PyMuPDF: ~2s para 100 páginas (vs PyPDF2 ~50s)
- python-docx: <1s para documentos típicos (<50 páginas)
